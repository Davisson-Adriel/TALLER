¿Qué es ECMAScript?


ECMAScript es un conjunto de normas que define cómo deben comportarse los lenguajes de programación como JavaScript. Es como un "manual de instrucciones" que asegura que el código funcione de manera consistente en todos los navegadores y entornos que lo soporten. Gracias a ECMAScript, se evita que el código tenga comportamientos inesperados cuando se ejecuta en diferentes plataformas.


Este estándar es mantenido por una organización llamada ECMA International, y su propósito es proporcionar una base común que garantice la interoperabilidad entre distintos navegadores, servidores y dispositivos. En otras palabras, ECMAScript asegura que todos los entornos que ejecutan JavaScript sigan las mismas reglas básicas para interpretar el código.

Versiones de ECMAScript

A lo largo de los años, ECMAScript ha tenido varias versiones que introdujeron nuevas características y mejoras. A continuación, se muestra un resumen de las versiones más importantes:

Versión Año Características Clave Relevante
ES1 1997 Primera edición de ECMAScript. Cimientos de ECMAScript
ES3 1999
  • Fue la versión que consolidó muchas de las bases del lenguaje, como la manipulación de cadenas, arrays, y objetos
  • Definió muchas de las funciones que todavía usamos hoy, como Array.prototype.push() y String.prototype.slice()
  • Introducción de las excepciones (try/catch)
 Estableció lo que es JavaScript hoy, pero no incluía características modernas como clases o funciones flecha
ES5 2009
  • Introducción del modo estricto (strict mode), que ayuda a evitar errores comunes y mejora la seguridad
  • Nuevas funciones para trabajar con arrays, como Array.prototype.forEach(), Array.prototype.map(), Array.prototype.filter()
  • Métodos como Object.defineProperty() para definir propiedades de objetos con configuraciones más precisas
  • Object.create() para crear objetos de una manera más segura
  • Mejoras en la manipulación de JSON, con JSON.parse() y JSON.stringify()
 Estableció nuevas formas de escribir código más seguro y limpio. Mejoró la seguridad y la eficiencia, pero aún carecía de muchas características modernas.
ES6 (ES2015) 2015
  • Introducción de clases en lugar de la función constructora tradicional
  • Permite dividir el código en módulos para facilitar la reutilización y el mantenimiento
  • let y const: Nuevas formas de declarar variables con alcance de bloque en lugar de var, lo que reduce errores de alcance
  • Facilita la gestión de operaciones asincrónicas, haciendo que el código sea más legible
  • Permite crear cadenas de texto con interpolación más fácil y legible (`Hola ${nombre}`)
  • Permite extraer valores de arrays u objetos de manera más fácil (const [a, b] = arr)
 Fue una de las actualizaciones más grandes de ECMAScript, proporcionando herramientas más poderosas y un código más limpio y modular
ES7 (ES2016) 2016
  • Operador de exponenciación (**): Permite elevar un número a una potencia de manera más sencilla (2 ** 3 en lugar de Math.pow(2, 3))
  • Array.prototype.includes(): Agrega una forma más fácil de comprobar si un array contiene un valor
 Aunque no fue tan grande como ES6, introdujo mejoras pequeñas pero útiles para trabajar con arrays y matemáticas
ES8 (ES2017) 2017
  • async y await: Proporciona una forma más sencilla y legible de trabajar con funciones asincrónicas, evitando el "callback hell" y haciendo el código más limpio y fácil de entender
  • Object.entries() y Object.values(): Métodos para trabajar de forma más directa con los valores y las entradas de un objeto
 async/await fue una de las mayores mejoras en la forma de manejar tareas asincrónicas, permitiendo escribir código que parece síncrono pero que en realidad es asíncrono
ES9 (ES2018) 2018
  • Spread/rest operator en objetos: Ahora puedes usar el operador ... también con objetos para hacer copias o combinarlos
  • Asincronía mejorada: Promise.finally() se agregó para que puedas ejecutar una acción una vez que una promesa se resuelva, sin importar si fue exitosa o fallida
  • RegEx mejorado: Nuevas características para trabajar con expresiones regulares, como el flag s (dotAll) que permite que . también coincida con saltos de línea
 Mejoras en la asincronía y en el manejo de objetos y expresiones regulares
ES10 (ES2019) 2019
  • Array.prototype.flat() y Array.prototype.flatMap(): Métodos para aplanar arrays y manipular arrays de manera más eficiente
  • Object.fromEntries(): Convierte un array de pares clave-valor en un objeto
  • String.prototype.trimStart() y trimEnd(): Métodos para eliminar espacios al principio o al final de una cadena
 Aumentó las herramientas para trabajar con arrays y objetos, facilitando la manipulación de datos
ES11 (ES2020) 2020
  • Nullish coalescing (??): Un operador que devuelve el valor de la derecha solo si el valor de la izquierda es null o undefined
  • Optional chaining (?.): Permite acceder a propiedades de objetos anidados de manera segura sin tener que verificar si el objeto intermedio existe
  • BigInt: Tipo de datos para trabajar con números enteros muy grandes
  • Promise.allSettled(): Permite manejar todas las promesas, independientemente de si se resuelven o rechazan
 Introdujo herramientas poderosas para manejar valores null o undefined y trabajar con grandes enteros (BigInt)
ES12 (ES2021) 2021
  • Método String.prototype.replaceAll(): Permite reemplazar todas las ocurrencias de una cadena en un texto
  • Logical Assignment Operators: Nuevos operadores como &&=, ||=, y ??= para hacer asignaciones condicionales de manera más limpia
 Introdujo mejoras en la manipulación de cadenas y operadores lógicos más compactos
ES13 (ES2022) 2022
  • Array.prototype.at(): Permite acceder a los elementos de un array con un índice negativo, similar a cómo lo hace Python. Esto facilita obtener el último elemento de un array, por ejemplo, con arr.at(-1)
  • Object.hasOwn(): Una nueva forma más segura y clara de comprobar si un objeto tiene una propiedad propia. Reemplaza a Object.prototype.hasOwnProperty(), evitando posibles errores si se sobrescribe este método
  • Top-level await: Permite usar await en el nivel superior de un módulo, sin necesidad de envolverlo en una función async. Esto mejora la legibilidad y simplifica el trabajo con código asincrónico en módulos
 Array.prototype.at() y el Top-level await son las mejoras más destacadas, facilitando el trabajo con arrays y simplificando el código asincrónico
ES14 (ES2023) 2023
  • Hashbang en archivos JavaScript: Permite el uso de #!/usr/bin/env node al principio de los archivos, una característica común en otros lenguajes de programación como Python. Esto hace que los archivos JavaScript se puedan ejecutar directamente desde la terminal sin necesidad de un comando explícito como node
  • Array.prototype.toSorted(): Añade la capacidad de crear una versión ordenada de un array sin modificar el original. Es una forma inmutable de ordenar los elementos
  • Method .at() para cadenas de texto (String.prototype.at()`): Así como en los arrays, ahora se puede acceder a caracteres dentro de una cadena utilizando índices negativos, lo cual facilita el trabajo con cadenas de texto
 Mejoras para el control de la recolección de basura con WeakRefs y un enfoque más seguro y flexible para trabajar con arrays y cadenas, como Array.prototype.toSorted() y String.prototype.at()

¿Como se Usa?

Herramientas Previas

Aprendizaje con ejemplos reales

1. let y const (ES6)


                    // Antes
                var nombre = "Laura";
                console.log(nombre);
                
                // Ahora
                let nombre = "Laura";
                const edad = 22;
                console.log(`${nombre} tiene ${edad} años.`);

2. Funciones flecha (ES6)


                    // Antes
                function saludar(nombre) {
                return "Hola " + nombre;
                }
                
                // Ahora
                const saludar = (nombre) => `Hola ${nombre}`;
                console.log(saludar("Pepe"));

3. async / await (ES8)


                // Antes: callbacks
                setTimeout(function () {
                console.log("Esperando...");
                }, 2000);
                
                // Ahora: async/await
                const esperar = () => new Promise(res => setTimeout(res, 2000));
                
                const cargar = async () => {
                console.log("Cargando...");
                await esperar();
                console.log("Listo!");
                };
                
                cargar();

4. Optional chaining y nullish coalescing (ES11)


                const persona = {};
                
                // Optional chaining
                console.log(persona.direccion?.calle); // undefined sin error
                
                // Nullish coalescing
                const apellido = null;
                console.log(apellido ?? "Apellido no proporcionado");

5. Array.at() (ES13)


                const frutas = ["manzana", "pera", "uvas"];
                
                // Antes
                console.log(frutas[frutas.length - 1]);
                
                // Ahora
                console.log(frutas.at(-1)); // uvas

6. Array.toSorted() (ES14)


                const numeros = [5, 3, 9, 1];
                
                // Ordena sin modificar el array original
                const ordenado = numeros.toSorted((a, b) => a - b);
                console.log("Original:", numeros);  // [5, 3, 9, 1]
                console.log("Ordenado:", ordenado); // [1, 3, 5, 9]
